JDK--安装及环境搭建 win10

今天试着搭建了win10 64位的JDK环境，好久没碰Windows系统了，几经周折，总算大功告成了。废话不多说了，先记录一下我搭建的过程和遇到的问题。
步骤：

  • 安装JDK + JRE
  • 配置环境变量
  • 检验JDK的安装

    JDK + JRE 安装

    想要安装JDK，首先需要有JDK的安装包，所以：
  1. 下载JDK
    点击JDK的官网下载地址
    根据电脑的系统(我的是win10、64位)，选择需要下载的JDK
  2. 安装JDK
    找到下载的JDK安装包，双击进入安装；
    根据安装提示弹窗的说明，一步步安装，注意安装的路径不！能！包！含！中！文！名！可以更改安装的目录，但是一定要记住自己安装的目录。我选择了默认的目录，直接安装。
  3. 安装JRE
    安装完JDK后，会自动提示你安装JRE的；
    根据安装提示弹窗的说明，一步步安装，同JDK的安装，可以更改安装的目录，但是不能选中JDK的安装目录，否则会覆盖掉JDK目录下的JRE目录的。我选择了默认的目录，直接安装。

    配置环境变量

    安装好了JDK和JRE之后，就开始环境变量的配置了：
  4. 我的电脑 → 右键 → 属性 → 高级系统设置 → 环境变量
  5. 新建一个变量名为“JAVA_HOME”的系统变量，变量值为“JDK的安装目录，根据个人所安装的目录修改”（我的变量值是：C:\Program Files\Java\jdk1.8.0_131）。注意：使用传统的“JAVA_HOME”环境变量名称，否则打开Android studio时可能会因为找不到JDK的路径报错。
  6. 再新建一个变量名为“CLASSPATH”，变量值为“.;%JAVA_HOME%\lib\dt.jar;%JAVA_HOME%\lib\tools.jar;”的系统变量，注意前面的点号和分号都是有的！
  7. 打开“Path”系统变量，点击新建，添加“%JAVA_HOME%\bin”和“%JAVA_HOME%\jre\bin”两个系统变量。（这一步添加变量到Path，使得系统可以在任何路径下识别java命令。）
  8. 点击“确认”保存环境变量的配置。

    检验JDK的安装

    配置完JDK的环境变量后，就开始检验了：
  9. 按Window+R，打开命令行窗口；
  10. 分别输入java和java -version和javac都正常运行即代表java已经正确安装，其中 java -version代表你安装的java的版本。
  11. 如果出现类似”java不是内部或外部命令”等提示语句：

    检查你的环境变量是否正确配置，如果有误，重新配置一遍;
    或者JDK和JRE安装目录是否重复，如果重复，可重新安装JDK。
